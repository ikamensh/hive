# Implementation notes

Architecture rationale lives in `wiki/architecture.md`; this file is the code map.

## Layout

- `hive/models.py` — pydantic domain models; persisted as dicts via the store. `Subscription` (AI plans the user owns) and `HumanTask` (operator todos with markdown instructions, e.g. CLI logins on remote runners) are org-level, not project-scoped. `OrchestratorRun` records one planner invocation's token usage + cost (the control-plane side of the bill; runner spend lives on `Task`).
- `hive/backends.py` — single source of truth for agent backends: the `REGISTRY` of `Backend(name, make_session)` records, derived `BACKEND_NAMES` (the orchestrator's allow-list, the runner's capability filter), the `make_session(backend, model)` dispatcher, and the usability-probe prompt (`PROBE_MARKER`, `probe_instructions`). Session factories import `kodo` lazily so the control plane imports the registry without kodo installed. Add a backend = one entry here.
- `hive/issues.py` — issues mode (`Project.work_source=issues`), a deterministic per-issue pipeline with **no orchestrator/LLM in the path** (full design in `wiki/issues-mode.md`). GitHub I/O (`fetch_open_issues_full` = issues+comments+embedded-image-URLs, `merge_branch` via the merges API, `resolve_issue_on_github`, authed like `github_repos.py`) is kept separate from the pure store ops (`reconcile`, `advance_issues`, `create_review_task`) so the latter test without network. Flow: a **human scan** (`POST /api/projects/{id}/scan-issues`) calls `reconcile` (open issue → `Workstream` source=issue status `queued`, full doc in `description`, image URLs in `issue_attachments`; cancel externally-closed; re-gate any still-open non-`done` issue with no live task — `blocked_clarity`/`rejected`/reopened/errored-mid-flight — back to `queued`), then `advance_issues` starts **strictly one issue at a time**: if none is in flight (`resolving`/`reviewing`) it promotes the lowest-`order` `queued` issue to `resolving` and queues one codex `gpt-5.5` **resolve** task on branch `hive/issue-<n>`. `advance_issues` is called again after every issue-task landing, so the next issue branches from a default branch that already has the prior fixes (avoids conflicts between issues that touch the same files). The resolve agent (`prompts/resolve.md`) clarifies (bug→fix by default; underspecified feature → `gh issue comment` + STOP) then fixes in the same warm session, ending `OUTCOME: FIXED|BLOCKED`. `api.task_result` parses it (`parse_resolve`): BLOCKED→`blocked_clarity`, FIXED→`reviewing` + queue a **review** task (`prompts/review.md`, fresh codex, fix-on-spot, ends `REVIEW: ACCEPT|REJECT`). `parse_review`: REJECT→`rejected` (agent commented), ACCEPT→`_land_review` merges the branch into the default branch (merges API, no PR) and closes the issue → `done` (merge/close failure escalates a `HumanTask`). The runner materializes `.hive/issue-<n>/` (ISSUE.md + downloaded attachments, git-excluded) before the agent runs (`runner.prepare_issue_workspace`). The supervisor skips the planner for issues projects (just dispatch + `compute_state`). Dormant ordered variant kept for reuse: `activate_next` (`queued`→`active`) + orchestrator `order_issues`/`resolve_issue` + `prompts/orchestrator_issues.md`. State: drained queue → `idle_no_open_issues`; any non-terminal issue with nothing in flight → `blocked_clarity` (waiting on a human).
- `hive/preflight.py` — issues-mode preconditions, so a misconfigured run fails up front instead of mid-pipeline. `preflight_checks` (control plane): `issues_mode`, `spec_repo_set`, `gh_token_present`, `repo_write_access` (the token's `permissions.push`, needed for merge+close), soft `issues_enabled`/`codex_runner_usable`. `create_preflight_task` queues a `TaskKind.preflight` runner self-check that `runner.run_preflight` executes against the real repo (push a throwaway branch + delete it, `gh auth status`) — the agent-facing git/gh auth the control plane can't see. `POST /api/projects/{id}/issues-preflight` (CLI `hive preflight`) runs the control-plane checks and, if they pass, queues+returns the runner check task; `scan-issues` re-runs the hard checks and 409s on failure, and reports `attachments_downloaded`/`attachments_failed`.
- `hive/escalation.py` — `escalate(store, title, instructions, project_id="")`: the "ask a human for help" primitive. Files a `HumanTask`, idempotent by (title, scope) so a per-tick recurring failure yields one todo. Used by the orchestrator-failure, spec-log-failure, and probe-login-failure paths.
- `hive/store.py` — `StoreBase` (ABC) is the contract; `MemoryStore` (tests), `FileStore` (local JSON under `HIVE_DATA_DIR/store`), and `FirestoreStore` (prod) are independent implementations. `update(model, id, mutate)` is the atomic read-modify-write — the way to mutate a contended doc (dispatch claim, result recording, resource counters, reboot requeue) without losing a concurrent writer: MemoryStore/FileStore guard every op with an RLock, FirestoreStore runs `mutate` in a transaction. `list(..., limit=N)` returns the most recent N (oldest→newest) so snapshot/`get_project` reads stay bounded (Firestore still sorts/slices in Python; pushing the slice down would need composite indexes). Plain `put` stays last-write-wins for fresh/uncontended writes. `hive/storage.py` selects the backend, exposes storage introspection, and copies a local store + blobs to GCP via `POST /api/storage/export`.
- `hive/supervisor.py` — deterministic layer: `compute_state` (pure, backend-aware — pending work whose backend no online runner offers is `blocked_resources`, not fake `working`), dispatch (serialized per repo; skipped when over the project's daily budget → `blocked_budget`), orphan failing, asyncio loop waking the orchestrator on events (incl. a replan nudge when work is stuck on an unavailable backend). `spend_today` sums both finished `Task.cost_usd` and today's `OrchestratorRun.cost_usd`, so a budget caps the planner's own LLM spend too — and the loop skips invoking the orchestrator entirely while over budget (it costs money), going quiet until the UTC-midnight rollover. Single-instance guard: a leader lease in the store (`claim_leader`, Firestore doc `settings/leader_lease`) is acquired at startup and renewed every tick, so a second control plane on the same Firestore refuses to boot (and a fenced-out one exits).
- `hive/llm/` — unified LLM access shared by every model-using flow. `core.py`: `ToolSet` (tool schemas from a callable's signature + docstring, and call dispatch), the `LLMAdapter` protocol (stateful `start`/`step`/`add_tool_results` conversation), and `ToolLoop` (provider-agnostic driver returning a `LoopResult` with the final text + summed `Usage` token counts both adapters report). `openai.py`/`gemini.py`: thin per-provider adapters — OpenAI is the manual chat-completions loop (+ model auto-select); Gemini drives `google-genai` function calling manually (auto-calling disabled) so the same loop owns the cycle. `provider.py`: `build_adapter(config)` + `resolve_provider`. `parsing.py`: `extract_json` (also used by critique). Adding a provider = one adapter; adding an agent role reuses `ToolLoop`.
- `hive/orchestrator.py` — wires the iteration loop to `hive/llm`: `_generate` builds the configured adapter (`HIVE_ORCH_PROVIDER=auto|openai|gemini`, `HIVE_ORCH_MODEL` optional) and runs `ToolLoop(MAX_REMOTE_CALLS)` over `ToolSet(Tools.functions())`; `_build_adapter` is the test-injection seam. Each `invoke` records an `OrchestratorRun` (`_record_cost`, priced via `hive/pricing.py`) so planner spend is visible and budgeted. `Tools` methods are the tool surface; docstrings are what the model sees. The quality gate is enforced here, not just prompted: verdicts are parsed deterministically in `api.task_result` (`parse_verdict`), `mark_goal_complete` is rejected unless every done workstream's latest task is an accepted verify, and `create_task` refuses a new work task after `MAX_FIX_ROUNDS` verify rejects without an accept (`_unresolved_rejects`) or `MAX_FIX_ROUNDS` consecutive failed work tasks with no success since (`_failed_work_streak`, the runaway-execution guard) — both forcing park+ask. PR-mode work goes on a per-workstream branch (`hive/<ws>`) so verify reviews exactly those changes; direct_push lands on the default branch with verify as an after-the-fact safety net. No `from __future__ import annotations` in this file — `ToolSet`/google-genai schema inference inspects the `Tools` methods' runtime annotations. Conversation history persists to the blob store; cold start is always safe because every invocation gets a full state snapshot + spec digest.
- `hive/pricing.py` — token-price estimates (USD/Mtok, longest-prefix match) that turn `OrchestratorRun` token counts into dollars; unknown models cost 0 (tokens still recorded). Refresh occasionally.
- `hive/specrepo.py` — shallow clone of the project's spec-home repo; digest for context (`digest_dir` works on any local dir), small commits for wiki/input-log distillation.
- `hive/critique.py` — spec critique (wiki/spec-critique.md): parallel critics (every model x tester/builder/consistency lens) + adjudicator run by the smartest model per `hive/model_intel.py` (Artificial Analysis index estimates). LLM transport is a `prompt -> text` callable (kept separate from the `hive/llm` adapters because `scripts/spec_critique.py` runs it via CLI agents, not API keys); JSON is parsed with `hive/llm`'s shared `extract_json`. Prompts: `hive/prompts/critic.md`, `adjudicator.md` (the "intake" role).
- `hive/api.py` — FastAPI: web API (unauthenticated; sits behind a tunnel/Tailscale) + runner protocol (shared token via `X-Hive-Token`). `create_app` takes the blob store so the runner can upload per-task JSONL traces (`POST/GET /api/tasks/{id}/trace`). `production_app()` wires env config; SPA fallback serves `web/dist`. Project create is name-only (draft); `PATCH` configures repos/toggles; `POST .../start` wakes the orchestrator with an optional mission/iteration brief.
- `hive/cli.py` — `hive` console script: full web-API parity, JSON output, for agents/scripts. `run(args, client)` takes any httpx-compatible client, so tests drive it with `TestClient` (`tests/test_cli.py` replays the whole loop CLI-side). `hive run`/`hive config` are the exceptions — they act locally instead of calling the API. `hive run` boots the control plane (uvicorn factory); `prepare_run_env` resolves tokens with precedence stored-config > ambient-env > `gh auth token` autodetect and prints provenance. `hive config set/unset/show/import` manages hive's own token store (`~/.config/hive/config.env`, chmod 600, keys allow-listed in `CONFIG_KEYS`) so the operator can give hive separate keys (e.g. separate cost tracking); stored values override ambient env on `run`, and `import` seeds the store from autodetected tokens.
- `hive/runner.py` — daemon: register → long-poll → checkout repo (the task's `branch` when set, else default) → run kodo `Agent` → upload the kodo JSONL trace → report. A watcher thread polls the task and calls `session.terminate()` on an operator cancel. Builds kodo sessions via `hive/backends.py` (claude/cursor/codex/gemini-cli) for cost parsing, timeouts, error classification.
- `hive/prompts/` — versioned base prompts (hash recorded on tasks for future GEPA).
- `web/` — React+Vite+TS SPA, polls the API every 4s (`usePoll` in `src/api.ts`; no state library). Pages in `src/pages/` (Projects, Project, Resources); markdown via `marked`; `VITE_MOCK=1` swaps the API client for in-memory fixtures (`src/mocks.ts`). Dev proxy `/api` → `:8000`; `npm run build` = `tsc --noEmit` + vite → `web/dist`. Project setup uses `RepoPicker` + shared `githubCatalog.ts`: one cached fetch, client-side filter; GitHub OAuth token on `User` drives catalog/validate (matches Hive login, not ambient `gh`); dev/`HIVE_GH_TOKEN` fallback when OAuth token absent.
- `deploy/` — Dockerfile (control plane), compose, GCE VM creation + startup script.
- `scripts/` — `smoke_orchestrator.py` (real-LLM smoke), `laptop_runner.sh`.

## Conventions

- Tests use `MemoryStore` + scripted orchestrator; no network, no LLM. Real-LLM behavior is checked with `scripts/smoke_orchestrator.py` (needs `OPENAI_API_KEY` or `GEMINI_API_KEY`).
- Secrets live in GCP Secret Manager (`hive-gemini-api-key`, `hive-gh-token`, `hive-runner-token`, `hive-openai-api-key`); the VM startup script materializes `/etc/hive/env`.
- GCP project `hive-ikamen`, Firestore `(default)` + bucket `hive-ikamen-blobs`, both `europe-west1`.

## Running

```bash
uv run pytest tests/                  # unit + mocked e2e
uvicorn --factory hive.api:production_app   # control plane (env: HIVE_GCP_PROJECT etc.)
python -m hive.runner                 # runner (env: HIVE_URL, HIVE_RUNNER_TOKEN)
bash deploy/create_vm.sh              # create/refresh the VM
bash scripts/laptop_runner.sh         # laptop runner via SSH tunnel
```
