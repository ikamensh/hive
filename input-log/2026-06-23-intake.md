The demo target is Hive itself. Either Codex or Claude is acceptable as the required scout/backend, as long as it is usable and trusted on the operator machine. Direct push is acceptable for the MVP proof; PR mode is not required for this iteration. Verified progress may be either a tested code change or a durable spec-only update, as long as Hive independently verifies or clearly records the result. GCP managed state remains the intended backend.

“Local demo” should not imply a separate product mode. It means the operator initiates the proof from this working environment, using Hive's existing local and/or distributed modes. The next work can self-answer implementation details while proving the path.
